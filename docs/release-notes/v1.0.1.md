# Apple Core 1.0.1

Apple Core 1.0.1 keeps remote MCP clients signed in after their 12-hour access token expires.

## OAuth refresh tokens

- OAuth clients now receive a refresh token with each access token. Apple Core rotates the refresh token every time it is used.
- Refresh tokens are bound to the client and MCP resource that received them. A token cannot be reused after rotation.
- Existing token files remain readable. Connections created before this release need one new sign-in because their saved records do not contain a refresh token.

This release also adds direct tests for refresh-token rotation, persistence, client and resource binding, and private token-file permissions.
