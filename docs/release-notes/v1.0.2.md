# Apple Core 1.0.2

Apple Core 1.0.2 fixes three issues that affected new-machine setup and adds control over the Dock icon.

## Fixes

- **Duplicate menu bar items**: the LaunchAgent now launches the app through `open -W` via LaunchServices instead of running the bare executable directly. LaunchServices deduplicates instances, so the LaunchAgent and a manual app open no longer produce two menu bar icons.
- **Service toggles not working on a new computer**: the app now fronts itself before requesting macOS privacy permissions. Without this, TCC silently denied permission requests from a background-only accessory app with no visible prompt, so toggling Calendar, Contacts, Reminders, or other services did nothing.
- **Sparkle update feed error**: the signed appcast was already corrected in 1.0.1; this release confirms the fix by shipping with the corrected feed configuration.

## New

- **Show Dock Icon toggle**: Apple Core is now a menu-bar-only app by default (no Dock icon). A new toggle in Settings > Server > General lets you opt into showing the Dock icon when the app is running.
